# built-in type

> 기본 타입과 `const` 한정자, 선언자 연산자로  만들 수 있는 타입으로
일반적인 컴퓨터 하드웨어의 능력을 직접적으로 그리고 효율적으로 가져다 쓴다.
> 
- 기본 타입
- const 한정자
- 선언자 연산자

# user-defined type

> C++ 추상 메커니즘을 사용해 여러 가지를 섞어 만든 타입으로
다른 말로 $클래스^{class}$와 $열거^{enumeration}$라고 한다.
사용자 정의 타입은 내장 타입과 다른 사용자 정의 타입으로 만들 수 있다.
> 

### 2.2 구조체

> 타입에 필요한 요소를 `struct` 라는 데이터 구조로 조직하는 것.
> 

#### stage 1

```cpp
struct Vector {
		double* elem;    // 원소들을 가리키는 포인터
		int sz;          // 원소 수
};
```

- 이때 v의 elem 포인터가 가리키는 것이 없어서 쓸모가 없는 상태이다.

---

#### stage 2

- v의 elem 멤버는 new 연산자가 생성한 포인터, v의 sz 멤버는 원소 수를 받아 초기화 하는 `vector_init()` 함수이다.
- `new` 연산자는 $자유\;공간^{free\;store}$($동적\;메모리^{dynamic\;memory}$와 $힙^{heap}$) 이라 부르는 영역의 메모리를 할당한다.
- 자유 공간에 할당된 객체는 그 객체가 생성된 범위와 독립적이며, `delete` 연산자로 소멸되기 전까지 살아있다(live).

---

#### stage 3

- 간단하게 Vector를 사용하는 방법이다.

---

#### `struct` 멤버에 접근할 때

```cpp
void f(Vector v, Vector& rv, Vector* pv) {
		int i1 = v.sz;      // 이름으로 접근
		int i2 = rv.sz;     // 참조로 접근
		int i3 = pv -> sz;  // 포인터로 접근
}
```

- 멤버에 접근하는 방법은 위와 같다.

### 2.3 클래스

> 타입의 인터페이스(누구나 접근하는 부분)와 구현(그 밖에 접근 불가능한 데이터에 접근하는 부분)을 구분해야 한다. 이를 수핸하는 언어 메커니즘을 $클래스^{class}$라고 부른다.
클래스는 $멤버^{member}$들의 집합이며, 데이터나 함수, 타입 멤버일 수 있다.
> 

#### 새로운 Vector 타입의 변수

> 
> 
> - 클래스의 인터페이스는 public 멤버로 정의된다.
> - private 멤버는 인터페이스로만 접근할 수 있다.
> - 클래스 선언에서 public과 private의 순서는 중요하지 않지만 표현을 강조하고 싶을 때를 제외하고는 관례상 public을 먼저 private를 나중에 선언한다.

```cpp
class Vector {
public:
		Vector(int s) :elem{new double[s]}, sz{s} { }  // Vector를 생성한다.
		double& operator[] (int i) { return elem[i]; } // 원소 접근: 첨자 지정(subscripting)
		int size() { return sz; }
private:
		double* elem;  // 원소들을 가리키는 포인터
		int sz;        // 원소 수
};

Vector v(6);       // 원소 6개로 된 Vector 정의
```

Vector 객체는 원소(elem)들을 가리키는 포인터, 원소 수(sz) 가 들어 있는 `핸들` 이기 때문에 객체 자체의 크기는 같다.

---

#### 2.2에서 만든 read_and_sum() 간소화 예제

- **기존의 코드**
- **간소화 예시**
    
    ```cpp
    class Vector {
    public:
    		Vector(int s) :elem{new double[s]}, sz{s} { }  // Vector를 생성한다.
    		double& operator[] (int i) { return elem[i]; } // 원소 접근: 첨자 지정(subscripting)
    		int size() { return sz; }
    private:
    		double* elem;  // 원소들을 가리키는 포인터
    		int sz;        // 원소 수
    };
    
    double read_and_sum(int s) {
    		Vector v(s);
    		for (int i = 0; i != v.size(); ++i) {
    				cin >> v[i];
    		}
    		
    		double sum = 0;
    		for (int i = 0; i != v.size(); ++i) {
    				sum += v[i];
    		}
    		
    		return sum;
    }
    ```
    
    위의 예시에서 볼 수 있는 것 처럼 public에 선언되어 있는 Vector 멤버 함수를 $생성자^{constructor}$라고 부르며 역할은 클래스 객체를 생성하는 함수이다. 
    Vector 타입의 객체를 어떻게 생성하는지 정의하였다.
    
    - 2.2에서 만들었던 `vector_init()` 대신 사용했다.
    - 원소 접근은 `operator[]`라는 $첨자^{subscript}$함수로 제공한다. 이 함수는 적절한 원소로의 참조(읽기, 쓰기 모두 허용하는 `double&` )를 반환한다.
    - `size()` 함수는 사용자에게 원소 수를 제공한다.

`struct` 와 `class` 는 기본적으로 차이가 없으며, `struct` 는 기본값으로 public 멤버를 갖는 class이다.

즉, struct에 생성자와 다른 멤버 함수를 정의할 수 있다.

### 2.4 열거

> 값을 열거할 수 있는 간단한 형태의 사용자 정의 타입
> 
- 예시
    
    #### stage 1
    
    ```cpp
    enum class Color { red, blue, green };
    enum class Traffic_light { green, yellow, red };
    
    Color col = Color::red;
    Traffic_light light = Traffic_light::red;
    ```
    
    - 이때 $열거자^{enumerator}$(앞선 코드 에서는 `red`)는 해당 `enum class` 의 범위에 속하기 때문에 다른 `enum class` 에 혼란 없이 반복적으로 쓰일 수 있다.
    - `enum` 뒤에 나오는 class는 열거가 $강형^{strongly-typed}$임을 명시하고 열거자의 범위를 지정한다. (scope 존재, 앞선 예시에서 열거자 red가 또 쓰일 수 있는 이유)
    
    ---
    
    #### stage 2
    
    ```cpp
    Color x1 = red;                   // 오류: 어떤 red인가?
    Color y2 = Traffic_light::red;    // 오류: 이 red는 Color가 아니다.
    Color z3 = Color::red;            // OK
    auto x4 = Color::red;             // OK: Color::red는 Color이다.
    
    int i = Color::red;               // 오류: Color::red는 int가 아니다.
    Color c = 2;                      // 초기화 오류: 2는 Color가 아니다.
    ```
    
    - 별도의 타입을 지정함으로써 `enum class` 는 우연히 상수를 잘못 사용하는 일이 없도록 방지한다.
    - Color와 정수 값도 암묵적으로 섞이지 않다.
    
    ---
    
    #### stage 3
    
    ```cpp
    Color x = Color{5}; // 컴파일 OK이지만 의미가 없다.
    ```
    
    - enum의 내부 타입(기본값은 int)값으로 enum을 초기화하려 하여 허용된다.
    
    ---
    
    ```cpp
    int x = int(Color::red);
    ```
    
    - enum값을 명시적으로 내부 타입으로 변환할 수도 있다.
    
    ---
    
    #### stage 4
    
    ```cpp
    Traffic_light& operator++(Traffic_light& t) { // 전위 증가 연산자: ++
    		switch (t) {
    		case Traffic_light::green: return t=Traffic_light::yellow;
    		case Traffic_light::yellow: return t=Traffic_light::red;
    		case Traffic_light::red: return t=Traffic_light::green;
    		}
    }
    
    auto signal = Traffic_light::red;
    Traffic_light next = ++signal; // next는 Traffic_light::green
    ```
    
    - 기본적으로 `enum class`에는 할당, 초기화, 비교만 정의돼 있다.
    - 하지만 사용자 정의 타입이기 때문에 열거에 사용할 연산을 정의할 수 있다.
    
    ---
    
    ```cpp
    Traffic_light& operator++(Traffic_light& t){ // 전위 증가 연산자: ++
    		using enum Traffic_light; // 여기서는 Traffic_light를 사용한다
    		
    		switch (t) {
    		case green: return t=yellow;
    		case yellow: return t=red;
    		case red: return t=green;
    		}
    }
    ```
    
    - `Traffic_light`라는 열거 이름을 반복해서 쓰기 싫다면 특정 범위 안에서 줄여 쓸 수 있다.
    
    ---
    
    ```cpp
    enum Color { red, green, blue };
    int col = green;
    ```
    
    - 여기서 `col` 값은 1로 열거자의 정숫값은 0부터 시작하고 1씩 증가한다.

### 2.5 공용체

> $공용체^{union}$는 모든 멤버를 같은 주소에 할당한 struct로서 그 union에서 가장 큰 멤버의 크기만큼 공간을 차지한다. union에는 1번에 딱 한 멤버의 값만 저장할 수 있다.
> 
- 예시
    
    이름과 값을 저장하는 기호표라면?
    
    ```cpp
    enum class Type { ptr, num }; // Type에는 ptr과 num 값을 저장할 수 있다.
    
    struct Entry {
    		string name;              // string은 표준 라이브러리 타입이다.
    		Type t;
    		Node* p;                  // t==Type::ptr이면 p를 사용한다.
    		int i;                    // t==Type::num이면 i를 사용한다.
    };
    
    void f(Entry* pe) {
    		if (pe->t == Type::num)
    				cout << pe->i;
    				// ...
    }
    ```
    
    - 이렇게 될 경우 멤버 p와 i를 동시에 저장할 일이 없어서 공간 낭비가 생기게 된다.
    
    #### 1. union을 사용
    
    ```cpp
    union Value {
    		Node* p;
    		int i;
    };
    ```
    
    - Value::p와 Value::i는 같은 Value 객체의 메모리 주소에 저장된다.
    
    ---
    
    ```cpp
    struct Entry {
    		string name;
    		Type t;
    		Value v; // t==Type::ptr이면 v.p를, t==Type::num이면 v.i를 사용한다.
    };
    
    void f(Entry* pe) {
    		if (pe-> t == Type::num)
    				cout << pe->v.i;
    		// ...
    }
    ```
    
    - 언어에서 별도로 union에 어떤 종류의 값이 저장되는지 관리하지 않는다.
    - 프로그래머가 직접 해줘야한다.
    
    $판별자^{discriminant}$나 $태그^{tag}$라고 부르는 $타입\,필드^{type\,field}$(예제에서는 t)와 union에 저장할 타입간 관련성을 유지하는 과정에서 오류가 발생하기 쉽다.
    
    - 공용체와 타입 필드를 클래스 내에 캡슐화해 관련시키고 올바르게 공용체를 사용하는 멤버 함수로만 접근하게 하면 오류를 막을 수 있다.
    - $태깅된\,공용체^{tagged\,union}$를 이용해 추상화하는 방법이 일반적이며 유용하다.
    
    #### 2. variant를 사용
    
    > 표준 라이브러리 타입을 이용하면 공용체를 직접 사용하지 않아도 된다.
    여러 타입 집합 중 하나의 값을 저장한다.
    > 
    
    ```cpp
    struct Entry {
    		string name;
    		variant<Node*,int> v;
    };
    
    void f(Entry* pe) {
    		if (hlods_alternative<int>(pe->v))     // *pe가 int를 저장하는가?
    				cout << get<int>(pe->v);           // int를 가져온다.
    		//...
    }
    ```
    
    많은 경우에서 union보다 variant가 사용하는 편이 더 간단하며 안전하다.
