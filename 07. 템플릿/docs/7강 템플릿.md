## 7.1 소개 - “왜 템플릿 인가?”

**템플릿(template)의 정의:**

> “템플릿은 타입 또는 값 집합으로 매개변수화한 클래스나 함수다.”
> 

`vector<double>` 처럼 원소 타입을 명시해서 특정 타입과 함수를 생성하도록 일반화한 개념이다.

---

## 7.2 매개변수화 타입

`double` 전용으로 만들었던 Vector를 템플릿으로 아래와 같이 일반화할 수 있다.

```cpp
template<typename T>
class Vector {
private:
    T* elem;  // T 타입 원소 배열
    int sz;
public:
    explicit Vector(int s);             // 생성자: 불변을 수립하고 자원을 획득한다.
    ~Vector() { delete[] elem; }        // 소멸자: 자원을 해제한다.

    // ... 복사와 이동 연산 ...

    T& operator[](int i);               // non-const Vector용
    const T& operator[](int i) const;   // const Vector용
    int size() const { return sz; }
};
```

멤버 함수 정의는 아래와 같이 할 수 있다.

```cpp
template<typename T>
Vector<T>::Vector(int s)
{
    if (s < 0) throw length_error{"Vector constructor: negative size"};
    elem = new T[s];
    sz = s;
}

template<typename T>
const T& Vector<T>::operator[](int i) const
{
    if (i < 0 || size() <= i) throw out_of_range{"Vector::operator[]"};
    return elem[i];
}
```

이렇게 만들면 아래와 같이 `Vector` 를 정의 할 수 있다.

```cpp
Vector<char>        vc(200);   // char 200개짜리 벡터
Vector<string>      vs(17);    // string 17개짜리 벡터
Vector<list<int>>   vli(45);   // list<int> 45개짜리 벡터
```

범위 기반 for 루프를 쓰려면  `begin()` 과 `end()` 을 아래와 같이 정의해야 한다.

```cpp
template<typename T>
T* begin(Vector<T>& x) { return &x[0]; }

template<typename T>
T* end(Vector<T>& x) { return &x[0] + x.size(); }
```

그러면 이렇게 작성할 수 있다.

```cpp
void write2(Vector<string>& vs)
{
    for (auto& s : vs)
        cout << s << '\n';
}
```

리스트, 벡터, 배열 등도 템플릿으로 정의할 수 있다. 템플릿은 컴파일 타임 메커니즘이다. `Vector<double>` 을 쓰면 컴파일러가 T자리에 double을 넣어서 코드를 생성하고 런타임 오버헤드 제로이다.

템플릿과 템플릿 인수 집합을 묶어 구체화, 특수화라 부른다. 컴파일 과정의 후반부인 구체화 타임 때 프로그램 내 각 구체화를 위한 코드를 생성한다.

### 7.2.1 제한된 템플릿 인수

> 그냥 `typename T` 로 열어두면 문제가 발생하게 된다. 예를들어 `Vector<thread>` 같은 걸 작성하게 되면 복사가 불가능한 `thread` 와 복사 연산을 제공하는 **Vector**. 컴파일 오류가 나게되고 에러메세지를 찾는데 어려움이 발생한다.
> 

그래서 아래와 같이 콘셉트로 인수를 제약한다.

```cpp
// typename 대신 Element라는 concept으로 제약
template<Element T>
class Vector {
private:
    T* elem;    // elem은 타입 T의 원소 sz개로 된 배열을 가리킨다.
    int sz;
    // ...
};
```

`template<Element T>` 는 T가 Vector에서 요구하는 특성을 만족하는지 검사하는 프레디킷이고, 이걸 콘셉트라고 부른다.

```cpp
Vector<int>    v1;  // OK: int는 복사 가능
Vector<thread> v2;  // 컴파일 타임 오류: 훨씬 명확한 메시지
```

이 예제에서 `Element` 는 표준 라이브러리 콘셉트인 `copyable` 정도면 충분하다. 콘셉트를 명시한 템플릿 인수를 **제한된 인수(contrained argument)**, 그 템플릿을 **제한된 템플릿(constrained template)**이라고 부른다.

콘셉트 검사는 완전히 컴파일 타임이다. 생성된 코드는 제한 없는 템플릿과 완전히 동일하다. 런타임 비용 또한 없다.

### 7.2.2 값 템플릿 인수

타입뿐만 아니라 정수 값도 템플릿 인수로 줄 수 있다.

```cpp
template<typename T, int N>
struct Buffer {
    constexpr int size() { return N; } // constexpr이게 컴파일 타임에 초기화가 
    T elem[N];  // 컴파일 타임에 크기 결정 -> heap 할당 없음
};

Buffer<char, 1024> glob;  // 전역: 정적 메모리에 char 1024개
void fct() {
    Buffer<int, 10> buf;  // 지역: 스택에 int 10개
}
```

`new` / `delete` 가 없으니 메모리 단편화도 없고 캐시 효율도 좋아서 자주쓰는 패턴이다.

참고로 문자열 리터럴은 기술적 이유로  값 템플릿 인수로 직접 쓸 수 없다. 필요할 때는 배열에 저장하여 쓴다.

```cpp
template<char* s>
void outs() { cout << s; }

char arr[] = "Weird workaround!";
// outs<"straightforward use">();  // 오류
outs<arr>();                       // 우회책
```

### 7.2.3 템플릿 인수 추론

템플릿의 구체화로 타입을 정의하려면 템플릿 인수를 명시해야 한다.

```cpp
// 예전 방식 — 타입 명시
pair<int, double> p = {1, 5.2};

// C++17 CTAD — 생성자 인수로부터 추론
pair p = {1, 5.2};  // pair<int, double>로 자동 추론
```

이렇게 많은 케이스에서 pair의 생성자가 초기자로부터 템플릿 인수를 추론할 수 있다.

이번에는 Vector를 이용한 예제이다.

```cpp
template<typename T>
class Vector {
public:
		Vector(int);
		Vector(initializer_list<T>);          // initializer-list constructor
		...
};

Vector v1 {1, 2, 3};            // Vector<int>로 추론
Vector v2 = v1;                 // Vector<int>로 추론

auto p = new Vector{1, 2, 3};   // p는 Vector<int>*이다.
Vector<int> v3(1);              // 여기서는 원소타입을 명시적으로 밝혀야 한다.

// 함정 케이스 
Vector<string> vs  {"Hello", "World"};    // 타입 명시
Vector         vs1 {"Hello", "World"};    // Vector<const char*>로 추론 (의외!)
Vector         vs2 {"Hello"s, "World"s};  // Vector<string>으로 추론 (s 접미사)
Vector         vs3 {"Hello"s, "World"};   // 오류: 원소 타입 혼재
Vector<string> vs4 {"Hello"s, "World"};   // 타입 명시했으니 OK
```

추론이 모호한 상황은 추론 안내(deduction guide)를 추가한다.

```cpp
template<typename Iter>
Vector(Iter, Iter) -> Vector<typename Iter::value_type>;

// 이제 반복자 쌍으로도 추론 가능
Vector v1 {1, 2, 3, 4, 5};
Vector v2(v1.begin(), v1.begin()+2);  // 반복자 쌍 -> Vector<int>로 추론
```

이 개념을 **CTAD(Class Template Argument Deduction)**이라고 부른다.

## 7.3 매개변수화 연산

템플릿은 컨테이너 매개변수화에만 쓰는 것이 아니다. 타입이나 값으로 매개변수화한 연산을 표현하는 세 가지 방법이 있다.

- 함수 템플릿
- 함수 객체: 데이터를 운반하는 객체로서 함수처럼 호출한다.
- 람다식: 함수 객체의 축약형

### 7.3.1 함수 템플릿

범위 기반 for로 순회할 수 있는 어떤 시퀀스든 원소 합을 구하는 함수:

```cpp
template<typename Sequence, typename Value>
Value sum(const Sequence& s, Value v)
{
    for (auto& x : s)
        v += x;
    return v;
}
```

`value` 인수를 따로 둔 이유 —  호출자가 어큐뮬레이터 타입과 초기값을 직접 지정하게 하기 위해서:

```cpp
Vector<int>             vi = {1, 2, 3, 4};
list<double>            ld = {1.1, 2.2, 3.3};
vector<complex<double>> vc = {{1,2}, {3,4}};

int    x  = sum(vi, 0);                    // Sequence=Vector<int>, Value=int
double d  = sum(vi, 0.0);                  // Value=double — 큰 int 합도 안전
double dd = sum(ld, 0.0);                  // double 리스트
auto   z  = sum(vc, complex{0.0, 0.0});   // complex 벡터
```

타입을 명시 안 해도 함수 인수에서 자동 추론된다.

주의할 점: 함수 템플릿은 `virtual` 멤버가 될 수 없다. 컴파일러가 프로그램 전체에 쓰인 모든 구체화를 알 수 없으니 vtbl을 생성할 수 없기 때문이다.

---

### 7.3.2 함수 객체

함수처럼 호출 가능한데 **상태(데이터)**를 가질 수 있다. 즉 일반 함수 포인터로는 못 하는 것.

```cpp
template<typename T>
class Less_than {
    const T val;  // 비교 기준값을 "저장"
public:
    Less_than(const T& v) : val{v} {}
    bool operator()(const T& x) const { return x < val; }
    //   ↑ 호출 연산자(application operator)
};

Less_than lti {42};           // 42보다 작은지 검사하는 객체
Less_than lts {"Backus"s};    // "Backus"보다 작은지 검사하는 객체
Less_than<string> lts2{"Naur"}; // 타입 명시 필요 (C 문자열이니까)

bool b1 = lti(30);   // true: 30 < 42
bool b2 = lts("Adams"s);  // true: "Adams" < "Backus"
```

이걸 알고리즘 인수로 쓰면 진짜 강력해진다.:

```cpp
template<typename C, typename P>
int count(const C& c, P pred)  // P는 프레디킷(predicate)
{
    int cnt = 0;
    for (const auto& x : c)
        if (pred(x)) ++cnt;
    return cnt;
}

void f(const Vector<int>& vec, const list<string>& lst, int x, const string& s)
{
    cout << count(vec, Less_than{x});  // x보다 작은 int 원소 개수
    cout << count(lst, Less_than{s});  // s보다 작은 string 원소 개수
}
```

**전역 함수 대신 functor를 쓰는 이유?**

비교 기준값을 전역변수 없이 전달할 수 있고, 인라이닝이 쉬워서 성능도 좋다. 이런 functor를 **정책 객체**(policy object)라고도 부른다.

### 7.3.3 람다식

`Less_than` 클래스를 따로 정의하고 쓰는 게 번거로울 때 람다는 그 자리에서 **익명 함수 객체**를 즉석으로 만드는 표기법이다.

```cpp
void f(const Vector<int>& vec, const list<string>& lst, int x, const string& s)
{
    // Less_than{x} 대신 람다로
    cout << count(vec, [&](int a){ return a < x; });
    //               캡처  인수      본문
    
    cout << count(lst, [&](const string& a){ return a < s; });
}
```

`[&](int a){ return a < x; }`는 `Less_than<int>{x}`와 비슷한 함수 객체를 생성한다.

#### 7.3.3.1 함수 인수로서의 람다

5장에서 `unique_ptr` 로 된 vector 원소들에 연산을 하려면 `draw_all()` 과 `rotate_all()` 처럼 함수를 여러개 작성해야 했다. 함수 객체(특히 람다)로 그 문제를 해결 할 수 있다.:

```cpp
template<typename C, typename Oper>
void for_each(C& c, Oper op)
{
    for (auto& x : c)
        op(x);  // 각 원소의 참조를 op()에 전달
}

// 이제 _all 함수 집합 필요 없이 한 번에 해결
void user()
{
    vector<unique_ptr<Shape>> v;
    // ...
    for_each(v, [](unique_ptr<Shape>& ps){ ps->draw(); });    // draw_all() 대체
    for_each(v, [](unique_ptr<Shape>& ps){ ps->rotate(45); }); // rotate_all(45) 대체
}
```

람다도 제네릭(generic)으로 만들 수 있다. `auto` 매개변수를 사용하면 된다.:

```cpp
template<class S>
void rotate_and_draw(vector<S>& v, int r)
{
    for_each(v, [](auto& s){   // auto -> 어떤 타입이든 OK, 즉 제네릭 람다
        s->rotate(r);
        s->draw();
    });
}

// unique_ptr<Shape>와 Shape* 컨테이너 모두에 동작
void user()
{
    vector<unique_ptr<Shape>> v1;
    vector<Shape*>            v2;
    rotate_and_draw(v1, 45);
    rotate_and_draw(v2, 90);
}
```

#### 7.3.3.2 람다로 초기화

조건에 따라 변수를 다르게 초기화해야 할 때 switch로 짜는 경우는 아래와 같다.:

```cpp
// 지저분한 패턴
vector<int> v;
switch (m) {
    case zero: v = vector<int>(n);  break;
    case cpy:  v = arg;             break;
    case seq:  v.assign(p, q);      break;
    // 문제점:
    // 1. v를 초기화 전에 쓸 수 있음
    // 2. case 하나 빠뜨려도 컴파일러가 못 잡음
    // 3. 초기화가 아니라 할당(assignment)임
}
```

람다를 초기자(initializer)로 쓰면 깔끔하게 해결된다.:

```cpp
// 좋은 패턴 — 람다를 초기자로
vector<int> v = [&] {
    switch (m) {
        case zero: return vector<int>(n);
        case seq:  return vector<int>{p, q};
        case cpy:  return arg;
    }
}();  // 즉시 호출(IIFE)

// case 빠뜨리면? 컴파일러가 return 없다고 경고
// v는 항상 완전히 초기화됨 (할당이 아닌 초기화)
```

#### 7.3.3.3 finally

소멸자는 더 이상 사용하지 않는 객체를 해제하는 암묵적 메커니즘인데, 소멸자가 없는 객체를 다룰 때가 있다. 이럴 때 범위가 종료될 때 필요한 동작을 실행하는 `finally()` 함수로 해결한다.:

```cpp
void old_style(int n)
{
    void* p = malloc(n * sizeof(int));  // C 방식 메모리
    auto act = finally([&]{ free(p); }); // 범위 종료 시 람다 자동 호출
    // ... 작업 ...
} // act 소멸 -> free(p) 자동 호출
```

`finally()` 의 구현은 아래처럼 간단하다.:

```cpp
template<class F>
[[nodiscard]] auto finally(F f)
{
    return Final_action{f};
}

template<class F>
struct Final_action {
    explicit Final_action(F f) : act{f} {}
    ~Final_action() { act(); }  // 소멸자에서 람다 호출
    F act;
};
```

`[[nodiscard]]`는 반환값을 변수에 저장하지 않으면 경고를 해준다.

→ 저장 안 하면 즉시 소멸해버리니까.

GSL(Core Guidelines Support Library)에서 `finally()`를 제공하고, 표준 라이브러리에서는 보다 정교한 `scope_exit` 메커니즘을 제안한다.

## 7.4 템플릿 메커니즘

좋은 템플릿을 정의하려면 4가지 언어 기능 지원이 필요하다.

- 타입에 따른 값: 변수 템플릿(Variable template) → 7.4.1절
- 타입과 템플릿의 에일리어스: 에일리어스 템플릿(alias template) → 7.4.2절
- 컴파일 타임 선택 메커니즘: `if constexpr` → 7.4.3절
- 타입과 표현식의 속성을 묻는 메커니즘: `requires` 식 → 8.2.3절

---

### 7.4.1 변수 템플릿

타입을 사용하려면 대개 상수와 그 타입의 값이 필요하다.

```cpp
template<class T>
constexpr T viscosity = 0.4;  // 타입마다 다른 상수

template<class T>
constexpr space_vector<T> external_acceleration = { T{}, T{-9.8}, T{} };

auto vis2 = 2 * viscosity<double>;      // double 버전
auto acc  = external_acceleration<float>; // float 버전
```

위의 코드는 유체 역학 시뮬레이션 예제이다. 대부분의 변수 템플릿이 상수로 보인다. 해당 예제는 타입 트레이트(type trait)와 조합하여 사용할 수 있다.

```cpp
template<typename T, typename T2>
constexpr bool Assignable = is_assignable<T&, T2>::value;
// is_assignable은 타입 트레이트(trait)이다(16.4.1)

template<typename T>
void testing()
{
    static_assert(Assignable<T&, double>, "can't assign double to T");
    static_assert(Assignable<T&, string>, "can't assign string to T");
}
```

표준 라이브러리에서도 `std::numbers::pi`, `std::numbers::log2e` 같은 수학 상수들이 변수 템플릿으로 제공된다.

### 7.4.2 에일리어스

`using` 으로 타입이나 템플릿의 별명을 만드는 것이다. 이식성과 가독성 두 가지 문제를 해결한다.

```cpp
// 이식성 — 구현마다 다를 수 있는 타입 숨기기
using size_t = unsigned int;
// (어떤 구현에서는 unsigned long일 수도 있음)
// 프로그래머는 size_t만 쓰면 이식 가능한 코드 작성 가능

// 템플릿 에일리어스 — 컨테이너 원소 타입 추출
template<typename C>
using Value_type = C::value_type;

template<typename Container>
void algo(Container& c)
{
    Vector<Value_type<Container>> vec;  // 컨테이너 원소와 같은 타입의 Vector
    // ...
}

// 부분 바인딩 — Map의 Key를 string으로 고정
template<typename Key, typename Value>
class Map { /* ... */ };

template<typename Value>
using String_map = Map<string, Value>;

String_map<int> m;  // Map<string, int>와 동일
```

모든 표준 라이브러리 컨테이너는 원소 타입의 이름으로 `value_type` 을 제공한다. 이 관례 덕분에 어떤 컨테이너에도 동작하는 제네릭 코드를 작성할 수 있다.

### 7.4.3 컴파일 타임 if

같은 함수에서 타입에 따라 다른 코드를 컴파일 타임에 선택한다.

```cpp
template<typename T>
void update(T& target)
{
    // ...
    if constexpr(is_trivially_copyable_v<T>)
        simple_and_fast(target);   // 단순 복사 가능한 타입 -> 빠른 경로
    else
        slow_and_safe(target);     // 복잡한 타입 -> 안전한 경로
}
```

일반 `if` 와 결정적인 차이는 아래와 같다.

> 
> 
> 
> 일반 if — 양쪽 브랜치 모두 컴파일됨
> → T가 slow_and_safe를 지원 안 하면 컴파일 오류 발생
> 
> if constexpr — 선택된 브랜치만 컴파일됨
> → T 타입에 맞지 않는 코드는 아예 컴파일 안 됨
> 

`is_trivially_copyable_v<T>` 는 T를 단순하게 복사할 수 있는지 알려주는 타입 프레디킷 이다.
